new_challenge(3, "Let's play TicTacToe")

Jhon Ramirez | Mon 28 August 2017

Hey you!

Today I'm going to introduce you my brand new challenge in my Android Course... it is a Tic-Tac-Toe game! :)

Here you can find the repo for this project or if you want you can download the source code from this link.

The challenge consists in follow this tutorial by Frank McCown (Harding University) and adapt it to newer Android versions, I'm going to use the API level 22 (Android 5.1 Lollipop), because it's the version installed in my Moto G (1st Gen), so I can test the app right into a physical device and see its behaviour and performance.

This tutorial was made a long time ago, and it works in devices with Android 2.2.x (Froyo) using the API level 8 onwards!!! so the GUI will be different.

The real challenge here is to adapt the existent functional logic to a newer and fresher user interface using recent versions of the Android API in such a way I can learn how the GUI works and how to use the widgets and the other elements in the screen to interact with the user inputs.

Then, the changes I made to tutorial are the following:

1. Menu

To add the game menu according to docs, first you have to create a XML file containing all the menu items.

Instead of building a menu in your activity's code, you should define a menu and all its items in an XML menu resource.

Here is my menu.xml

<?xml version="1.0" encoding="utf-8"?>
<menu xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:android="http://schemas.android.com/apk/res/android">

    <item
        android:id="@+id/start_new_game"
        android:title="@string/star_new_game_label" />
    <item
        android:id="@+id/quit"
        android:title="@string/quit_label" />
</menu>

To specify the options menu for an activity, override the onCreateOptionsMenu() method.

@Override
public boolean onCreateOptionsMenu(Menu menu) {
    MenuInflater inflater = getMenuInflater();
    inflater.inflate(R.menu.menu, menu);
    return true;
}

Finally, you just have to handle the user selection, to do that you have to override the onOptionsItemSelected() method in the activity. Following is how I did it.

public boolean onOptionsItemSelected(MenuItem item) {
    switch (item.getItemId()) {
        case R.id.start_new_game:
            startNewGame();
            return true;
        case R.id.quit:
            Toast.makeText(getApplicationContext(), "See you soon!", Toast.LENGTH_SHORT).show();
            handler.postDelayed(new Runnable() {
                @Override
                public void run() {
                    finish();
                    System.exit(0);
                }
            }, 2000);
        return true;
            default:
    return super.onOptionsItemSelected(item);
    }
}

2. Delay

Another change I made was to add a delay when Android plays, because it is so fast that you never see the update in the information textview. So i think it could be more "human" if you see that the android is "thinking" its next move. To add a delay you can use a handler, I did it following this answer.

private final Handler handler = new Handler();
(...)
handler.postDelayed(new Runnable() {
    @Override
    public void run() {
        int move = mGame.getComputerMove();
        setMove(TicTacToeGame.COMPUTER_PLAYER, move);
        mThinking = false;
        updateWinner(mGame.checkForWinner());   
    }
}, 2000);

3. Toast

I like toasts! So I added some user notifications using Toast. It's easy to do it, you only have to import android.widget.Toast; and using it like this:

Toast.makeText(getApplicationContext(), "See you soon!", Toast.LENGTH_SHORT).show();

4. Borderless

This was the last change, I only made it because of the "Buttons in button bars should be borderless;" warnings in the Android Studio. The borderless buttons are for cohesiveness of design throughout the android platform. It's recommended you follow the guidelines so that the user feels that your app is native and does not stick out like a sore thumb.

<Button style="@style/Widget.AppCompat.Button.Small"
    android:id="@+id/one"
    android:layout_width="100dp"
    android:layout_height="100dp"
android:textSize="70sp" />
